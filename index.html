Layer-Wise Coordination between Encoder and Decoder for Neural Machine Translation

Introduction

本文提出了神经机器翻译分层协调(layer-wise coordiantion)的概念，其逐层协调encoder、decoder每层隐层表征的学习。具体来说，本文设计了layer-wise attention和mixed attention机制，同时共享了encoder、decoder每层的参数权重。

Motivation

目前最佳的神经机器翻译(Transformer)采用了一种端到端的结构，其由encoder与decoder两部分组成。其中，encoder、deocder都是stack of N layers的结构，decoder利用encoder的最后一层输出做为输入来生成各个目标位置的隐层表征。
对于此，本文作者有如下疑惑：

Why should the low-level representation of a target token base on the highest-level ones of the source token?
Why not attenting each layer of the decoder to each responding layer of the encoder?

The Model

Layer-wise coordination


Layer-wise coordination对Transformer结构做了两方面的修改，即decoder中的每层需要attends to encoder中相对应的每一层、attention以及FFN层参数权重的共享。该模型的encoder、decoder拥有相同的层数，encoder中$i$层source token的隐层表征基于self-attention处理$(i-1)$层所有的source tokens而获得。公式如下：
$$S_{t}^{i} = selfAttn(s_{1}^{i-1}, s_{2}^{i-1}, …, s_{T}^{i-1})$$
其中，$S$表示source token的隐层表征，$T$表示source sequence的长度，$i$表示模型的$i-th$层。
该模型的decoder利用一种mixed attention机制来处理$i$层所有source tokens的隐层表征以及$(i-1)$层前$t$个target tokens的隐层表征。公式如下：
$$H_{t}^{i} = mixedAttn(s_{1}^{i}, s_{2}^{i}, …, s_{T}^{i}, H_{1}^{i-1}, …, H_{t-1}^{i-1})$$
其中，$H$表示target token的隐层表征。

Mixed attention

mixed attention的实现比较简单，它只是在原始dot-product attention中增加了mask信息，来防止模型attending到非法的位置。具体公式如下：
$$Mixed_attention(Q,K,V) = softmax(\frac{QK^{T}}{\sqrt{d_{model}}} + M) V$$
$$M(i,j)=
\begin{cases}
0& \text{j < n V j =< i+n}\
-infty& \text{otherwise}
\end{cases}$$

贡献

提出了layer-wise coordination的概念，实现了mixed attention;

共享每层参数权重，可以保证encoder、decoder堆叠更多的层，模型变瘦、拉长;

Neural Machine Translation by jointly Learning to Align and Translate

Introduction

现有的神经网络翻译模型大多数是基于encoder-decoder框架的，其利用encoder将输入序列编码到一个固定长度的向量中，decoder依据此编码向量生成输出序列。传统的encoder-decoder框架中固定长度的编码向量是提升seq2seq模型性能的瓶颈，本文作者首次将注意力机制引入到了NLP领域。

The Model

RNN Encoder-Decoder


在编码端，编码器将给定的输入序列$X = X_{1},x_{2},…,x_{T_{x}}$编码得到context vector $c$中，具体公式如下:
$$h_{t}=f(x_{t},h_{t-1})$$
$$c = q({h_{1},h_{2},…,h_{T_{x}}})$$
其中，$h_{t}\in R^{n}$是$t$时刻的隐层状态，$f,q$是非线性方程。大多数情况下，我们会设置$f$为bi-lstm，$c = h_{T_{x}}$。


在解码端，首先我们需要计算$i$时刻的隐层状态，其依赖于当前时刻的输入$x_{i}$与上一时刻的隐层状态$s_{i-1}$。
$$s_{i} = f(x_{i},s_{i-1})$$
然后依据$i$时刻的隐层状态$s_{i}$、上一时刻的输出$y_{i-1}$以及context vector $c$来预测当前时刻的输出。
$$p(y_{i}) = p(y_{i}|y_{1},y_{2},…,y_{i-1}, c) = g(y_{i-1},s_{i},c)$$
整个输出序列$Y$的概率，计算如下：
$$p(Y) = \prod_{i=1}^{T}{p(y_{i}|y_{1},y_{2},…,y_{i-1}, c)}$$

Learning to align and translate


在Seq2seq-Attention模型结构中，我们重新定了$i$时刻解码端的隐层状态$s_{i}$以及该时刻输出值的条件概率：
$$s_{i} = f(y_{i-1},s_{i-1},c_{i})$$
$$p(y_{i}) = p(y_{i}|y_{1},y_{2},…,y_{i-1}, c) = g(y_{i-1},s_{i},c_{i})$$
对于$c_{i}$的计算如下：
$$e_{it} = align(s_{i-1}, h_{t})$$
$$a_{it} = \frac{exp(e_{it})}{\sum_{k=1}^{T_{x}}{exp(e_{ik})}}$$
$$c_{i} = \sum_{t=1}^{T_{x}}{a_{it}h_{t}}$$
其中，$T_{x}$表示输入序列的长度，文本对齐函数align被定义为一个FFN，但其常见的定义也包括：dot、concat、general，具体公式如下：
$$align(s_{i-1},h_{t}) = {s_{i-1}}^T h_{t} \qquad dot $$
$$align(s_{i-1},h_{t}) = {s_{i-1}}^T W_a h_{t} \qquad dot general $$
$$align(s_{i-1},h_{t}) = W_a [s_{i-1}; h_{t}]) \qquad concat $$

贡献

首次将注意力机制引入NLP任务中;

seq2seq with attention在机器翻译任务中表现优异;

不足

seq2seq with attention的编码端、解码端仍然采用了RNN(lstm、bi-lstm)，无法并行，速度受限；

编码器与解码器的链接仅仅依靠一个固定大小的状态向量，使得解码器无法关注输入信息更多的细节；

Sequence to sequence learning with Neural Networks

Introduction

此论文提出了一种端到端的序列学习方法，其利用多层LSTM把输入序列映射到一个固定维度的向量中，然后用另外一个多层LSTM网络对编码向量解码得到目标序列。

The model

RNN

输入序列：$(x_1, x_2, x_3, …, x_T)$，输出序列：$(y_1, y_2, y_3,…, y_T)$，标准RNN通过迭代公式(1)来计算序列的隐层状态：
\begin{equation}
h_t = sigmoid(W^{hx} x_t + W^{hh} h_{t - 1})
\end{equation}
$t$时刻输出序列的预测如下：
$$y_{t} = W^{yh}h_{t}$$
其中，$h_{t}$是$t$时刻的隐层状态，$W^{hx},W^{hh},W^{yh}$分别是相应的权重矩阵, $t$时刻的RNN如下图：

seq2seq

最基础的seq2seq模型包含三个部分，即编码器、解码器、中间状态变量。编码器通过多层LSTM将输入序列映射到固定维度的向量中，解码器通过多层LSTM对编码向量进行解码得到目标序列。 假设输入序列：$(x_1, x_2, x_3, …, x_m)$，输出序列：$(y_1, y_2, y_3,…, y_n)$，编码端隐层状态：$(s_{1}^{i},s_{2}^{i},…, s_{m}^{i})$，解码端隐层状态：$(h_{1}^{j}, h_{2}^{j},…,h_{n}^{j})$。编码端、解码端都采用了多层的LSTM，编码端隐层状态计算如下：
$$s_{t}^{i} = g(s_{t}^{i-1},s_{t-1}^{i} )$$
其中，$s_{t}^{i}$是$i^{th}$层$t$时刻的隐层状态，$s_{t}^{0} = x_{t}$，$g$是激活函数。对于解码端，其起始位置用特殊字符<EOS>做标识：
$$h_{t}^{i} = g(y_{t}^{i}, y_{t-1}^{4},h_{t-1}^{i} ) \qquad i = 1 $$
$$h_{t}^{i} = g(h_{t}^{i-1},h_{t-1}^{i} ) \qquad 1 < i < 4 $$
$$y_{t}^{4} = softmax(h_{t}^{4}) \qquad i = 4 $$


此图来源于 cs224n Lecture1，原文中并没有给出多层LSTM的公式，上述公式是根据此图推得。 目标语句的条件概率如下：
$$p(y_1, y_2, …, y_{n} | x_1, x_2, …, x_m) = \prod_{t =1}^{n}{p(y_t | v, y_1, y_2, ..y_{t-1})}$$
其中$v$是编码向量。

贡献

使用了两个不同的多层LSTM，一个用于处理输入序列，一个用于处理输出序列;

深层的LSTM性能优于浅层的LSTM;

颠倒输入序列的顺序能提升模型性能;

不足

编码器将输入编码为固定大小的状态向量，其是一个信息有损压缩的过程，会出现信息丢失的问题；

随着序列长度的增加，时间维度上的序列很长，RNN模型也会出现梯度弥散问题；

编码器与解码器的链接仅仅依靠一个固定大小的状态向量，使得解码器无法关注输入信息更多的细节；
